###DATOS_PERSONA###

- grupo_muscular_favorito: array of favorite muscle groups (prioritize these)
- train_experience: user's experience level (beginner, intermediate, advanced, expert)
- Other fields: weight_kg, height_cm, age, injuries, pathologies, diseases

2. Exercise distribution per day: approximately 70% of exercises from grupo_muscular_favorito, 30% from the complementary group. Target 5 to 6 exercises per day.
3. The appropriate repetitions, sets, and rest times for each exercise using the reference table provided in ###VOLUMEN###.
4. Only select exercises from the table ###EJERCICIOS### matching the selected muscle groups for that day.

###DIAS###
This is a JSON array where each entry has a "date" (YYYY-MM-DD) and a "day" (weekday name in English).

###VOLUMEN###

- Each day must have a DIFFERENT category combination from the other days. Spread the user's grupo_muscular_favorito across the available training days.

- Approximately 70% of exercises per day should come from grupo_muscular_favorito, 30% from the complementary group.
- Select exercises ONLY from the category(ies) assigned to that specific day.
- Use the reference table (###VOLUMEN###) to adjust repetitions, sets, and rest times per muscle group.
- Sets per exercise (Series): beginner = 2 to 3 sets, intermediate = 3 to 4 sets, advanced or expert = 4 to 5 sets. Use the ###VOLUMEN### table only as reference for weekly volume — do not use those numbers directly as sets per exercise.
- Rest time ("Descanso"): 2 minutes for large muscle categories, 1.5 minutes for small muscle.
- Repetitions: use the rep range from ###VOLUMEN### adjusted to the user's train_experience (higher reps for beginners, lower reps for advanced).
- Number of exercises per day based on train_experience: beginner = 4 to 5, intermediate = 5 to 6, advanced or expert = 6 to 7.

Output format:
Return a flat JSON array where each element is one training day. The array must contain EXACTLY one object per date in ###DIAS###, in the same order.

Each day object must have:
  - "fecha": date as YYYY-MM-DD (exactly as provided in ###DIAS###)
  - "semana": 1 (always 1, since this is the template week)
  - "categorias": array of muscle category names for that day (must match categories in ###EJERCICIOS###)
  - "nombre": string joining the categories with " + " (e.g., "PECHO + TRÍCEPS")
  - "ejercicios": array of exercises, each with:

          - "Descanso": rest time in minutes
          - "Detalle series": array with one object per set, each with "Reps" (number) and "carga" (0 if unknown)
